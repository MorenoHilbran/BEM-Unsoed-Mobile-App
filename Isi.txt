Siap! Berikut rancangan ringkas MVP (Minimum Viable Product) untuk Mobile App BEM UNSOED sesuai struktur halaman yang kamu minta.
1) Ruang Lingkup MVP (3 fitur utama)
F1. Informasi Event Besar
* Lihat event besar terdekat (tanggal, lokasi, poster, CTA daftar).


F2. Socmed Post / Blog
   * Feed linimasa (post teks + foto 1–4).

   * Interaksi dasar: Like, Komentar, Report.

   * Buat Post (tulis, upload 1–4 foto, tag topik: #event, #info, #opini).

F3. Info Kost
      * Daftar kost sekitar kampus (harga/bulan, alamat, jarak, fasilitas, kontak WA).

      * Filter cepat: harga, jarak, gender, fasilitas.

      * CTA “Hubungi via WhatsApp”.

________________


2) Arsitektur Navigasi (3 halaman)
Tab 1 – Beranda
         * Banner/Poster “Gensoed Merch Pamflet” (CTA ke link merch).

         * Kartu Event Besar Terdekat (1 teratas, swipe untuk 3-5 event berikutnya).

         * Shortcut grid: Podcast, Komik, E-Magz (link ke menu/halaman lain).

Tab 2 – Feed (Socmed Post Blog)
            * Feed postingan user lain (avatar, nama, prodi/angkatan opsional, waktu).

            * Aksi: Like, Komentar, Report.

            * Composer mengambang “+” untuk buat post.

Tab 3 – Profil
               * Header profil: foto, nama, NIM (opsional), bio singkat.

               * Tab mini: Post Saya • Tersimpan • Pengaturan.

               * Aksi: Edit profil, Keluar.

________________


3) User Stories & Acceptance Criteria (ringkas)
US-E1 (Event):
 Sebagai mahasiswa, saya melihat event besar terdekat di Beranda agar cepat daftar.
✓ Beranda menampilkan ≥1 event aktif terurut tanggal terdekat.
✓ Tombol “Daftar/RSVP” & “Ingatkan” berfungsi.
US-P1 (Feed):
 Sebagai user, saya membaca feed dan memberi like/komentar.
✓ Feed memuat 10 post terbaru, infinite scroll.
✓ Like/komentar realtime (optimistic update).
✓ Report tersedia (alasan preset).
US-P2 (Create Post):
 Sebagai user, saya membuat post dengan teks + foto.
✓ Validasi: teks 1–500 karakter; 0–4 foto (≤2MB/each).
✓ Setelah kirim, post muncul di urutan teratas.
US-K1 (Kost):
 Sebagai mahasiswa, saya mencari kost dengan filter harga & jarak.
✓ Filter bekerja; klik kost membuka detail + tombol Chat WA.
US-PR1 (Profil):
 Sebagai user, saya mengubah profil (nama/bio/foto).
✓ Perubahan tersimpan & tampil di feed/post.
________________


4) Peran & Moderasi Konten (MVP)
                  * Guest: baca event & kost; tidak bisa posting.

                  * User (SSO/Email kampus): semua fitur + posting.

                  * Moderator/Admin BEM: hapus/ban post, sematkan event.

                  * Moderasi: Report → daftar antrean admin → tindakan (hapus/suspend).

________________


5) Model Data (minimal)
users(id, name, email, role[guest|user|admin], avatar_url, major, year, bio, created_at)
events(id, title, start_at, end_at, location, cover_url, organizer, reg_url, price, status[draft|published], is_highlight)
event_rsvps(user_id, event_id, status[going|interested], notified_at)
posts(id, user_id, content, images_json[], topics[], created_at)
post_likes(post_id, user_id, created_at)
post_comments(id, post_id, user_id, content, created_at)
post_reports(id, post_id, user_id, reason, created_at, handled_by, handled_at, action)
kosts(id, name, price_month, address, lat, lng, distance_km, contact_wa, facilities[], photos[], gender_rule, verified, rating_avg)
merch(id, title, image_url, link_url, active_from, active_to, is_active)
Catatan: arrays bisa disimpan sebagai JSON (images_json, facilities) atau di-normalisasi bila butuh skala.

6) API Minimal (contoh REST)
                     * GET /events?highlight=1&limit=5

                     * POST /events/{id}/rsvp body: {status}

                     * GET /posts?limit=10&cursor=…

                     * POST /posts body: {content, images[]}

                     * POST /posts/{id}/like, POST /posts/{id}/comment

                     * POST /posts/{id}/report body: {reason}

                     * GET /kost?max_price=&max_dist=&gender=&facility=wifi

                     * GET /me, PATCH /me (update profil)

7) Desain UI Ringkas per Halaman
Beranda
                        * Hero Banner (Gensoed Merch): gambar, judul, CTA.

                        * Section Event: Card besar (tanggal, lokasi, tombol RSVP/Ingatkan).

                        * Shortcut 3 ikon: Podcast • Komik • E-Magz.

Feed
                           * Kartu post: avatar, nama, waktu, teks, 0–4 foto (grid), aksi Like/Comment/Report.

                           * FAB “+” untuk create post.

Profil
                              * Header: foto bulat, nama, bio, stats (Post, Tersimpan).

                              * List post milik user; tombol Edit profil; tombol Keluar.

________________


8) Notifikasi (MVP)
                                 * Reminder event (H-3 & H-1 untuk RSVP “going”).

                                 * Notif komentar baru pada post saya.

                                 * Promo merch aktif baru (maks 1/minggu).

________________


9) Metrik Keberhasilan (MVP)
                                    * Event: CTR kartu event ≥8%, konversi RSVP ≥3%.

                                    * Feed: DAU feed, avg likes/post, 7-day retention.

                                    * Kost: CTR “Chat WA” ≥10%, apply filter ≥30% sesi.

________________


10) Rencana Rilis (2 Sprint x 2 minggu)
Sprint 1: Auth (SSO/email), Event highlight di Beranda, RSVP+Reminder, Feed baca, Model data dasar.
Sprint 2: Create post (foto), Like/Comment/Report + Moderasi, Info Kost + filter + CTA WA, Profil edit, Notifikasi dasar.
________________


11) Catatan Teknis (opsi cepat)
                                       * Frontend: Flutter atau React Native (Expo) + bottom tabs.

                                       * Backend: Laravel 12 (sesuai ekosistemmu) + MySQL/Postgres; atau Supabase untuk percepatan.

                                       * Media: S3/Cloud Storage, limit 2MB/foto (compress di klien).

                                       * Login: SSO email kampus (@mhs.unsoed.ac.id) bila memungkinkan.
